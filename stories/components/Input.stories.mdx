# Input

The `'input'` component is one of the only classes which is designed with a specific element in mind: `<input>`.

<Canvas>
  <Story name="Default">
    <input className="input" placeholder="Input" />
  </Story>
</Canvas>

## Labeled

`'input-label'` and `'input-hint'` can be used to provide additional context to the input.

The `'input-group'` class can be wrapped around the input and these elements to help with layout and positioning.

<Canvas>
  <Story name="Labeled">
    <label className="input-group">
      <p className="input-label">Oranges</p>
      <input className="input" placeholder="Your opinion?" />
      <p className="input-hint">They're a (subjectively) delicious fruit.</p>
    </label>
  </Story>
</Canvas>

## Disabled

The `'muted'` class can be added to the input's label to style it when its input is disabled.

<Canvas>
  <Story name="Disabled">
    <label className="input-group">
      <p className="muted">Disabled</p>
      <input className="input" disabled placeholder="Disabled" />
    </label>
  </Story>
</Canvas>

## Validation

### Error

`'input-error'` and `'input-success'` classes can be *added* to the element making use of `'input'` to mark the input's value, or lackof,
errorful or successful.

<Canvas>
  <Story name="Error">
    <label className="input-group">
      <p className="input-label">Error</p>
      <input className="input input-error" placeholder="An errorful empty input" />
    </label>
  </Story>
</Canvas>

<Canvas>
  <Story name="Success">
    <label className="input-group">
      <p className="input-label">Success</p>
      <input className="input input-success" placeholder="A successfully empty input" />
    </label>
  </Story>
</Canvas>

## Floating Label

Add the `'floating'` class to an `'input-group'` to create a floating label.

> Due to the way [adjecent sibling selectors](https://developer.mozilla.org/en-US/docs/Web/CSS/Adjacent_sibling_combinator) work, `'input-label'` must appear *after* `'input'`.

> Your `'input'` element *will* need a `placeholder` assigned.

<Canvas>
  <Story name="Floating Label">
    <label className="input-group floating">
      <input autoComplete="name" className="input" placeholder="Jane Doe" />
      <p className="input-label">Full Name</p>
      <p className="input-hint">This hint doesn't float :(</p>
    </label>
  </Story>
</Canvas>

As expected, manually adding a `value` to the `'input'` will cause its label to appear in the "floating" position:

<Canvas>
  <Story name="Floating Label w/ Value">
    <label className="input-group floating">
      <input autoComplete="name" className="input" placeholder="Jane Doe" value="Janeth Doeith" />
      <p className="input-label">Full Name</p>
      <p className="input-hint">This hint doesn't float :(</p>
    </label>
  </Story>
</Canvas>

## Floating Validation

Both `'input-error'` and `'input-success'` validation classes can be re-used to style a floating input.
They should be *added* to the `'input'` element, just as they would for a traditional input field.

<Canvas>
  <Story name="Floating Error">
    <label className="input-group floating">
      <input className="input input-error" placeholder="Floating" />
      <p className="input-label">Floating Error</p>
      <p className="input-hint">This hint doesn't float :(</p>
    </label>
  </Story>
</Canvas>

<Canvas>
  <Story name="Floating Success">
    <label className="input-group floating">
      <input className="input input-success" placeholder="Floating" />
      <p className="input-label">Floating Success</p>
      <p className="input-hint">This hint doesn't float :(</p>
    </label>
  </Story>
</Canvas>
