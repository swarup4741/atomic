# Input

The `input` component is (one of) the only class(es) which is designed with a specific markup in mind.
That said, it is still perfectly acceptable to re-use its styling in other styles; Doing say may include unnecessary styling, depending on your application's needs.

The simpliest input can simply use the `'input'` class, providing very simple border, placeholder, and text styling.

<Canvas>
  <Story name="Default">
    <form>
      <input className="input" placeholder="Input" />
    </form>
  </Story>
</Canvas>

## Labeled

`'text'` and `'hint'` can be used to provide additional context to the input.

The `'input-group'` class can be wrapped around the input and these elements to provide a consistent spacing.

<Canvas>
  <Story name="Labeled">
    <form>
      <label className="input-group">
        <p className="text">Oranges</p>
        <input className="input" placeholder="Your opinion?" />
        <p className="hint">They're a (subjectively) delicious fruit.</p>
      </label>
    </form>
  </Story>
</Canvas>

## Disabled

The `'muted'` class can be added to the input's label to style it when its input is disabled.

<Canvas>
  <Story name="Disabled">
    <form>
      <label className="input-group">
        <p className="muted">Disabled</p>
        <input className="input" disabled placeholder="Disabled" />
      </label>
    </form>
  </Story>
</Canvas>

## Floating Label

* `'input-floating'`
* `'input-floating-group'`
* `'input-floating-label'`

> **Note:** Due to the way TailwindCSS works, the `'peer'` class *must* be added to the element which uses `'input-floating'`.
> The `'input-floating-label'` element must appear *after* the `'input-floating'` element.

<Canvas>
  <Story name="Floating Label">
    <form>
      <label className="input-floating-group">
        <input className="input-floating peer" placeholder="Floating" />
        <p className="input-floating-label">Floating</p>
        <p className="hint">This hint doesn't float :(</p>
      </label>
    </form>
  </Story>
</Canvas>
