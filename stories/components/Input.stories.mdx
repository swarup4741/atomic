# Input

The `input` component is (one of) the only class(es) which is designed with a specific markup in mind.
That said, it is still perfectly acceptable to re-use its styling in other styles; Doing say may include unnecessary styling, depending on your application's needs.

The easiest way to use the input is through the `'input'` class, providing very simple styling to the input's border, placeholder, and text.

<Canvas>
  <Story name="Default">
    <form>
      <input className="input" placeholder="Input" />
    </form>
  </Story>
</Canvas>

## Labeled

`'text'` and `'hint'` can be used to provide additional context to the input.

The `'input-group'` class can be wrapped around the input and these elements to help with layout and positioning.

<Canvas>
  <Story name="Labeled">
    <form>
      <label className="input-group">
        <p className="text">Oranges</p>
        <input className="input" placeholder="Your opinion?" />
        <p className="muted hint">They're a (subjectively) delicious fruit.</p>
      </label>
    </form>
  </Story>
</Canvas>

## Disabled

The `'muted'` class can be added to the input's label to style it when its input is disabled.

<Canvas>
  <Story name="Disabled">
    <form>
      <label className="input-group">
        <p className="muted">Disabled</p>
        <input className="input" disabled placeholder="Disabled" />
      </label>
    </form>
  </Story>
</Canvas>

<Canvas>
  <Story name="Disabled with a value">
    <form>
      <label className="input-group">
        <p className="muted">Disabled</p>
        <input className="input" disabled placeholder="Disabled" readOnly value="Hello World" />
      </label>
    </form>
  </Story>
</Canvas>

## Validation

### Error

`'input-error'` and `'input-label-error'` can be *added* to the respective input and label to provide a visual indication of an error.

<Canvas>
  <Story name="Error">
    <form>
      <label className="input-group">
        <p className="input-label-error">Error</p>
        <input className="input input-error" placeholder="Error" />
      </label>
    </form>
  </Story>
</Canvas>

### Success

`'input-success'` and `'input-label-success'` can be *added* to the respective input and label to provide a visual indication of a successful input.

<Canvas>
  <Story name="Success">
    <form>
      <label className="input-group">
        <p className="input-label-success">Success</p>
        <input className="input input-success" placeholder="Success" />
      </label>
    </form>
  </Story>
</Canvas>

## Floating Label

The following classes can *replace* their respective counterparts to create a floating input.

* `'input-floating'`
* `'input-floating-group'`
* `'input-floating-label'`

> **Note:** Due to the way [adjecent sibling selectors](https://developer.mozilla.org/en-US/docs/Web/CSS/Adjacent_sibling_combinator) work, `'input-floating-label'` must be added *after* `'input-floating'`.

<Canvas>
  <Story name="Floating Label">
    <form>
      <label className="input-floating-group">
        <input className="input-floating" placeholder="Floating" />
        <p className="input-floating-label">Floating</p>
        <p className="muted hint">This hint doesn't float :(</p>
      </label>
    </form>
  </Story>
</Canvas>

## Floating Validation

Both the `'input-*'` and `'input-label-*'` validation classes can be used to style a floating input.

<Canvas>
  <Story name="Floating Error">
    <form>
      <label className="input-floating-group">
        <input className="input-floating input-error" placeholder="Floating" />
        <p className="input-floating-label input-label-error">Floating</p>
        <p className="muted hint">This hint doesn't float :(</p>
      </label>
    </form>
  </Story>
</Canvas>

<Canvas>
  <Story name="Floating Success">
    <form>
      <label className="input-floating-group">
        <input className="input-floating input-success" placeholder="Floating" />
        <p className="input-floating-label input-label-success">Floating</p>
        <p className="muted hint">This hint doesn't float :(</p>
      </label>
    </form>
  </Story>
</Canvas>

## Disabled Floating Label

Unlike the default label, the `'input-floating-label'` element will provide its own styling for the `disabled` state.

  <Story name="Disabled Floating Label">

      <label className="input-floating-group">
        <input className="input-floating" disabled placeholder="Floating" />
        <p className="input-floating-label">Floating</p>
      </label>
    </form>
  </Story>
</Canvas>
